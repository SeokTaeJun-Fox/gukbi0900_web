> position의 정의
   - static : 일반적인 문서의 흐름에 따라 배치한다 (defalut)
   - relative : 각 요소들을 하나로 겹치게 한 뒤 z-index를 사용해야 할 때
   - absolute : 부모의 relative 요소를 찾은 뒤 해당 테두리 영역 내에서 움직이도록 한다.
   단 absolute로 요소를 겹치게 하면 신경 쓸 것이 더 많아진다 
   - fixed : 문서상의 흐름을 무시하고, 최상위 조상 요소를 기준으로 잡는다.
   - sticky : 문서의 스크롤을 따라오게 만든다.
       단, 부모 요소에 overflow 속성이 있다면 사용이 불가능하다.

> tranfrom이란?
   - 자기 자신의 요소를 어떻게 배치할 지 정의할 수 있는 속성
   
   translate()
      - 위치를 조정하는 속성 [%, px]

   scale()
      - 요소의 크기를 조절하는 속성 (0~1)

   rotate()
          - rotateX
          - rotateY
    
      % == turn (0~1)
      각도 == deg (degree 디그리)

   skew()
      - 비틀기를 조절하는 속성
      ex) skew(15deg, 15deg)

   여러개 전달이 가능하다
   transfrom: scale(0.5) translate(100px, 100px)

> 미디어 쿼리
   - 단말기의 유형에 따라 화면에 해상도가 다르므로,
   사용자가 보고 있는 뷰 포트에 따라서 다른 유형의 화면(레이아웃)을 보여주는 쿼리

미디어 쿼리 문법
   @media [media-type] and ([media-feature-rule]) {
      
   }

미디어 타입의 조건
   and: 둘 다 만족하면 true
   or: 둘 중 하나라도 만족하면 true
   not: true를 false로 false를 true로

미디어 타입의 종류
   all, braille, embossed, handheld, print, projection, screen, speech, tty, tv
