프로그램
   실행되지 않은 상태.

프로세스
   실행된 프로그램.

쓰레드 (JS는 싱글쓰레드)
   - 프로세스 처리 경로
   - 비동기식 방식으로 특정 연산을 수행할 때 업무를 효율적으로 처리하기 위해 
   짧은 시간안에 다양한 연산들을 마치 동시에 처리하는 것처럼 하는 것들을 할 수 있게 하는 것
   - 쿨타임, 유튜브를 보고 있을 때, 무한스크롤 등등
   
   - 단일 쓰레드
      처리 경로를 한 개만 가지고 있기 때문에 직렬적이다.
      동시에 많은 양을 처리하기 힘들기 때문에 상대적으로 비효율적이다.
      하지만 하나의 작업에 문제가 발생하더라도 다른 작업에는 영향을 끼치지 않는다.
      따라서 안정성이 보장되고 설계 시 멀티 쓰레드에 비해 쉽다.

   - 멀티 쓰레드
      하나의 프로세스를 동시에 처리하는 것처럼 보이지만 사실은 매우 짧은 단위로 분할해서 차례로 처리한다.
      여러 개의 처리 경로를 가질 수 있도록 하며, 동시 작업이 가능해진다.
      설계하기 굉장히 어려우며, 하나의 쓰레드 문제 발생 시 모든 쓰레드에 문제가 발생하게 된다.
      멀티 쓰레드로 설계했다면, 처리량 증가, 효율성 증가,
      처리비용 감소의 장점이 있기 때문에 단점을 감수하고 설계하는 편이다.

동기(Synchronous) 프로그래밍
   - 작업을 순차적으로 처리하는 방식
   앞의 결과가 끝나야 다음 작업을 시작한다.

동기의 장점
   - 디버깅이 상대적으로 쉽다

동기의 단점
   - 시간이 오래 걸리는 작업이 있으면 전체 흐름이 멈춘다
   ex) 네트워크/웹 통신 I/O, DB I/O, 파일 I/O 등..
   사용자의 경험이 나쁠 수 있다.

비동기(Asynchronous) 프로그래밍
   - 작업을 백그라운드에서 처리하고(JS에서 리버브), 나중에 결과가 도착하면 알림을 받는다
   기다리지 않고 다음 작업을 바로 실행한다

비동기의 장점
   - 긴 작업 중에도 다른 작업을 동시에 수행할 수 있음.
   자원을 더 효율적으로 사용 가능 (특히 I/O 작업에 적합)

비동기의 단점
   - 코드 흐름이 복잡해질 수 있음(콜백 지옥, Promise 체인, aysnc/await 등)
   디버깅이 어렵다.

비동기의 예시
   - 데이터를 요청하는것
   - 몇초마다, 몇초뒤에 코드
   - 파일, 이미지
